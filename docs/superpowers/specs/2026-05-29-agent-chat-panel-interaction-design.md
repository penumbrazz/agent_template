# Agent Chat Panel 前端交互效果

## 背景

前端需要实现一个类似 Laper 写作页面中 Agent 对话面板的交互效果。本阶段只覆盖桌面端的前端容器交互，不接真实后端，不实现完整 AI 对话能力。消息、会话列表和助手回复均使用 Mock 数据或本地状态。

目标是把 Chat Panel 的状态切换、拖动、右侧停靠、宽度调整、历史会话抽屉和新对话行为描述清楚，方便后续前端实现时对齐体验。

## 范围

本阶段包含：

- Chat Panel 的最小化、浮动展开、右侧停靠三种状态
- 浮动状态下的面板拖动
- 右侧停靠状态下的宽度调整和宽度记忆
- 面板内部的历史对话抽屉
- 基于 Mock 数据的新对话、历史会话切换和简单输入

本阶段不包含：

- 后端 API 对接
- 流式 AI 回复
- 真实会话持久化
- 权限、计费、模型选择等业务逻辑

## 状态设计

### 最小化状态

页面右下角显示一个固定的胶囊按钮，例如「Agent」或「AI Assistant」。按钮浮在主界面之上，不占用页面布局空间。

点击胶囊按钮后，Chat Panel 进入浮动展开状态。胶囊按钮需要包含图标和文字，并提供稳定的 `data-testid`。

### 浮动展开状态

浮动面板默认出现在右下角附近，和浏览器右边、底边保留固定间距。面板是独立浮层，包含标题栏、内容区和底部输入区。

标题栏作为拖动区域。用户可以按住标题栏拖动面板到浏览器视窗内的任意位置。

拖动必须受到视窗边界限制：面板的上、下、左、右边缘都不能被拖出浏览器可视区域。尤其不能让标题栏离开视窗，否则用户会失去再次拖回面板的操作入口。浏览器窗口尺寸变化时，也需要重新校正面板位置，保证面板仍然完整留在视窗内。

浮动位置不持久化。刷新页面，或从最小化状态重新打开时，面板回到默认右下角附近。

### 右侧停靠状态

用户点击停靠或最大化按钮后，Chat Panel 停靠到页面最右侧，成为固定右栏。右栏右边贴住浏览器边缘，高度覆盖主要工作区。

右侧停靠状态下，面板左边框是可拖拽的 resize handle。用户按住左边框向左拖动时，聊天区域变宽；向右拖动时，聊天区域变窄。

停靠宽度需要有最小值和最大值限制，例如最小 `360px`，最大不超过视口宽度的一定比例或固定上限，避免聊天区过窄或过度挤压主内容。

用户调整后的停靠宽度需要持久化到浏览器本地状态。下次进入右侧停靠状态时，恢复上次调整后的宽度。

## 状态切换规则

- 从最小化状态点击胶囊按钮，进入浮动展开状态。
- 从浮动展开状态点击最小化按钮，回到右下角胶囊按钮。
- 从浮动展开状态点击停靠按钮，进入右侧停靠状态。
- 从右侧停靠状态点击还原按钮，回到浮动展开状态。
- 从右侧停靠状态点击最小化按钮，回到右下角胶囊按钮。
- 从最小化状态再次打开时，默认进入浮动展开状态，不自动恢复为右侧停靠状态。
- 如果面板被最小化，历史对话抽屉应同步关闭。下次展开时默认不显示历史抽屉。

## 历史对话抽屉

Chat Panel 标题栏左侧提供三横杠菜单按钮。点击后，在 Chat Panel 内部从左侧展开历史对话抽屉。该抽屉不是全局页面导航，不应影响主页面布局。

历史抽屉参考效果是：左侧浅奶油面板滑出，右侧聊天内容区域被半透明遮罩压暗。

历史抽屉内容包括：

- 顶部标题「历史对话」
- 新对话图标按钮
- 返回或收起按钮
- 会话列表

每个会话列表项显示：

- 会话标题
- 消息条数
- 更新时间

当前会话项需要高亮，例如使用浅色背景。

点击历史会话后，切换到对应 Mock 会话内容，并关闭历史抽屉。点击抽屉外遮罩、返回按钮或收起按钮时，只关闭历史抽屉，不改变当前会话。

历史抽屉必须跟随当前面板模式：

- 在浮动展开状态下，历史抽屉只在浮动面板内部展开，不超出面板边界。
- 在右侧停靠状态下，历史抽屉只在停靠面板内部展开，不影响主页面布局。

## 新对话行为

标题栏或历史抽屉里的「+」或「新对话」按钮用于创建新的 Mock 会话。

点击新对话后：

- 当前聊天内容清空，或切换到欢迎占位状态。
- 当前会话标题显示为「新对话」。
- 历史列表中新增或激活一个「新对话」条目。
- 如果历史抽屉正在打开，点击新对话后应关闭抽屉，回到主聊天界面。
- 新会话只存在于前端 Mock 状态中，不调用后端 API。

输入第一条消息后，可以继续保持标题为「新对话」，也可以使用消息前几个字生成 Mock 标题。具体标题生成不是本阶段重点。

## 面板内部内容

本阶段只实现容器交互所需的最小聊天内容：

- 少量 Mock 历史会话
- 少量 Mock 消息
- 一个输入框
- 一个发送按钮

用户在输入框中发送文本后，可以在本地追加一条用户消息，并可选追加一条固定 Mock 助手回复。该行为只用于验证容器和会话切换效果，不代表真实 AI 能力。

## 视觉约束

样式需要遵守项目 `DESIGN.md`：

- 默认画布使用奶油色背景。
- Chat Panel 使用浅奶油或卡片表面色。
- 主操作按钮使用珊瑚色。
- 按钮、输入框圆角约 `8px`。
- 面板圆角可使用 `12px`。
- 图标优先使用 `lucide-react`。

所有交互元素必须添加稳定、描述性的 `data-testid`，包括：

- 最小化胶囊按钮
- 标题栏菜单按钮
- 停靠按钮
- 还原按钮
- 最小化按钮
- 历史抽屉关闭按钮
- 新对话按钮
- 历史会话列表项
- 输入框
- 发送按钮
- 右侧停靠 resize handle

## 验收标准

- 能从右下角胶囊按钮打开 Chat Panel。
- 能从浮动展开状态收起为右下角胶囊按钮。
- 浮动面板可以通过标题栏拖动。
- 浮动面板拖动时，任何边缘都不会离开浏览器视窗。
- 浏览器窗口尺寸变化后，浮动面板仍完整保留在视窗内。
- 能从浮动展开状态切换到右侧停靠状态。
- 右侧停靠状态下，面板贴住浏览器右边缘。
- 右侧停靠状态下，左边框可拖动调整宽度。
- 停靠宽度受最小值和最大值限制。
- 调整后的停靠宽度刷新后仍能恢复。
- 能从右侧停靠状态还原为浮动展开状态。
- 三横杠菜单按钮能打开历史对话抽屉。
- 历史对话抽屉只影响 Chat Panel 内部，不影响主页面布局。
- 点击历史会话项后，能切换到对应 Mock 会话并关闭抽屉。
- 点击新对话按钮后，能创建或切换到新的 Mock 会话。
- 面板最小化后，历史对话抽屉状态会关闭。
- 所有关键交互元素都有稳定 `data-testid`。
