Create an Alembic migration for: $ARGUMENTS

## Steps

1. Read the existing models in `backend/app/models/` to understand current schema
2. Read recent migrations in `backend/alembic/versions/` for naming and style conventions
3. Generate migration file in `backend/alembic/versions/` with:
   - Timestamp prefix (auto-generated by Alembic)
   - Descriptive revision message
   - `upgrade()` function with the schema change
   - `downgrade()` function that reverses the change
4. Validate by running: `cd backend && uv run alembic check` (verify no drift)

## Conventions

- Use `op.create_table`, `op.add_column`, `op.alter_column` etc.
- Include `sa.Column` with proper types and nullable settings
- Always include a reversible `downgrade()`
- Use `op.create_index` for new foreign keys or queried columns
- Reference existing models via imports for consistency
